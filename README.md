Portfolio Website

Website portfolio pribadi saya yang dibangun untuk mendukung personal branding sebagai Web Developer(Laravel).

Repository ini berfungsi sebagai frontend portfolio yang terintegrasi dengan Supabase untuk manajemen data proyek, sertifikat, dan sistem komentar realtime.

🎯 Purpose

Website ini dibuat untuk:

Menampilkan proyek backend secara terstruktur

Menyediakan sistem manajemen data berbasis Supabase

Mendukung update project tanpa perlu mengubah source code

Membangun personal branding sebagai backend-oriented developer

🛠 Tech Stack
Frontend

React

Tailwind CSS

Framer Motion

AOS (Animate On Scroll)

Lucide Icons

Backend (Portfolio Data Layer)

Supabase (PostgreSQL Database)

Row Level Security (RLS)

Supabase Realtime

Supabase Storage

🏗 Architecture Overview

Frontend bertugas sebagai presentation layer.

Supabase digunakan sebagai:

Database untuk tabel projects, certificates, dan portfolio_comments

Storage untuk menyimpan gambar

Realtime engine untuk sistem komentar

Struktur ini memungkinkan:

Penambahan project tanpa deploy ulang

Pengelolaan data langsung dari Supabase dashboard

Skalabilitas sederhana untuk portfolio dinamis

📂 Core Database Structure
projects
CREATE TABLE public.projects (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamp with time zone DEFAULT now() NOT NULL,
  "Title" text,
  "Description" text,
  "Img" text,
  "Link" text,
  "Github" text,
  "Features" jsonb,
  "TechStack" jsonb
);

RLS diaktifkan dengan kebijakan read-only public access untuk kebutuhan portfolio.

🚀 Local Development Setup
1. Clone Repository
git clone https://github.com/Hammam-GNF/portfolio.git
cd portfolio
2. Install Dependencies
npm install

Jika terjadi peer dependency issue:

npm install --legacy-peer-deps
3. Setup Environment Variables

Buat file .env di root project:

VITE_SUPABASE_URL=your-supabase-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key

Pastikan:

Semua variabel diawali VITE_

.env tidak di-commit ke repository

4. Run Development Server
npm run dev

Akses melalui:
http://localhost:5173

🏗 Production Build
npm run build

File hasil build berada di folder dist dan siap untuk di-deploy ke:

GitHub Pages

Vercel

Netlify

Shared hosting

🔐 Security Notes

Row Level Security (RLS) diaktifkan pada semua tabel

Public hanya memiliki akses SELECT

Insert dibatasi sesuai kebutuhan (misalnya sistem komentar)

Supabase anon key digunakan hanya untuk akses terbatas
